1.2. Etapas

Puede resumirse el desarrollo de aplicaciones web como el proceso de creación de un programa informático (es decir, software) para realizar las diferentes tareas que cubren una necesidad. Estas necesidades pueden haberse generado en cualquier parte de una organización; sin embargo, todas comparten una necesidad común: automatización y aumento de la eficiencia.

Cada proceso de creación de aplicaciones sigue los mismos pasos:

1. Planificación y análisis de requisitos

Es una de las etapas más importantes. La realizan los miembros más experimentados del equipo junto con el cliente, En esta se planifica el enfoque básico del proyecto, se estudia su viabilidad económica, operativa y técnica. Además, se identifican los rasgos asociados para minimizarlos todo lo posible.

2. Definición de requisitos

Definir y documentar claramente los requisitos y obtener la aprobación del cliente. Estos requisitos son los que se diseñarán e implementarán durante el ciclo de vida del desarrollo software.

3. Diseño de la arquitectura

Se aborda la solución desde diferentes puntos de vista hasta encontrar el enfoque que se considera óptimo. Este enfoque de diseño debe definir claramente todos los módulos junto con su comunicación y representación. El diseño interno de cada uno de los módulo debe estar definido con el más mínimo detalle. Además, debe incluir unos estándares de calidad que se desean alcanzar.

4. Construcción o implementación

Si las fases anteriores se han ido realizando correctamente, la generación del código debería realizarse sin dificultad. Aquí se utilizan herramientas de programación como compiladores, intérpretes, depuradores, etc. La elección del lenguaje de programación depende del tipo de software que se desarrolle.

5. Pruebas

Los defectos del producto de informan, rastrean, reparan y vuelven a probar, hasta que el producto alcanza los estándares de calidad definidos en el diseño de la arquitectura.

6. Despliegue y mantenimiento

Se despliega la aplicación en producción, es decir, se "saca al mercado". Luego, se corrigen los defectos detectados y se realizan todas las modificaciones necesarias para que el producto sea más eficiente. A partir de aquí, se pueden realizar modificaciones sobre todas las demás etapas.

Este es el modelo común, aunque según las políticas de cada organización las fases pueden dividirse o compactarse para ajustarse mejor a las particularidades de cada sector.
